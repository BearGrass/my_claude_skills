# 小说结构修订助手

对小说初稿进行深度结构分析和系统性修订。

## 触发前提

此技能应在以下情况使用：
- 用户已完成初稿（`05-小说初稿/`）
- 用户想要对作品进行结构性修改

## 工作流程

1. **通读分析**：阅读全文，标注问题
2. **问题分类**：按优先级和类型整理问题
3. **修订方案**：针对每个问题提出修改方案
4. **逐章修订**：执行修订并记录
5. **验证效果**：确认问题已解决

## 问题分析框架

### 问题优先级评估

```
致命问题（必须修改）：
- 情节逻辑漏洞
- 人物行为严重不合理
- 主题严重偏离

严重问题（应该修改）：
- 节奏明显拖沓
- 人物弧线断裂
- 结局仓促或无说服力

轻微问题（可以修改）：
- 个别对话不够自然
- 少量场景可精简
- 细节描写可优化
```

### 修订策略矩阵

```
| 问题类型 | 修复方式 | 工作量 |
|----------|----------|--------|
| 情节漏洞 | 补充场景/调整因果 | 高 |
| 节奏拖沓 | 删减/合并 | 中 |
| 人物问题 | 增加铺垫/调整动机 | 中-高 |
| 逻辑矛盾 | 修改设定/补充解释 | 低-高 |
| 结局问题 | 增加铺垫/重写 | 高 |
```

## 修订工作流程

### 阶段一：全局阅读

```
1. 完整阅读全文，不做修改
2. 记录每个问题点
3. 标注章节位置
4. 不纠结细节，把握整体
```

### 阶段二：问题整理

```
1. 将问题按类型分类
2. 评估每个问题的严重性
3. 确定修订优先级
4. 估算工作量
```

### 阶段三：方案制定

```
1. 针对每个问题提出方案
2. 评估方案的连锁影响
3. 确定最优方案
4. 制定修订顺序
```

### 阶段四：执行修订

```
1. 从致命问题开始
2. 每个修订记录在案
3. 修订后检查连锁影响
4. 更新问题清单状态
```

### 阶段五：效果验证

```
1. 重新阅读修订部分
2. 确认问题已解决
3. 检查是否引入新问题
4. 如有问题，继续迭代
```

## 常见问题与解决方案

### 情节漏洞

```
问题：主角突然知道一个他不可能知道的信息
解决：
1. 增加获取信息的场景
2. 改变信息来源
3. 调整主角的知情范围
```

### 节奏拖沓

```
问题：中间几章进展缓慢
解决：
1. 删减无推进作用的场景
2. 合并相似场景
3. 增加新的冲突元素
4. 提高信息密度
```

### 人物崩坏

```
问题：人物行为与之前设定矛盾
解决：
1. 增加行为转变的铺垫
2. 调整行为本身
3. 增加内心挣扎的描写
4. 通过其他人物反应来合理化
```

### 结局仓促

```
问题：结局来得太快太突然
解决：
1. 增加高潮前的铺垫章节
2. 在前文埋下更多伏笔
3. 减缓高潮前的节奏
4. 增加结局后的收束场景
```

**输出**：完整的修订记录和修订稿
