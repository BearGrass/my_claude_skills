# 小说创作工作流引导器

引导用户完成从灵感到成品的完整小说创作流程，全程遵循创作规范。

## 规范基础

**所有创作活动均遵循 `novel-standards` 规范**，包括：
- 文件目录：作品存放在 `~/novels/` 目录
- 写作风格：网文风格，适合短剧/动画/有声书改编
- 文字校对：自动检查错别字和标点符号
- 内容边界：适度情感描写，避免低俗
- 文笔标准：简洁有力，画面感强

详见 `novel-standards` 技能。

## 工作模式

### 模式一：完整引导（推荐新手）
1. 规范确认 → 项目创建 → 灵感创意 → 背景设定 → 人物设定 → 故事骨架 → 写作控制台 → 质量验收 → 输出成品

### 模式二：快速启动
1. 获取基本信息 → 在 ~/novels/ 创建项目 → 从指定步骤开始

### 模式三：继续创作
1. 检测 ~/novels/ 下的项目 → 读取进度 → 继续创作

### 模式四：直接写作
1. 确认有大纲 → 直接进入 novel-writing

## 执行流程

### 步骤0：规范确认

```
┌─────────────────────────────────────────────────────────┐
│                    创作规范确认                          │
├─────────────────────────────────────────────────────────┤
│  📁 文件存放：~/novels/                                  │
│  ✍️ 写作风格：网文风格，适合改编                         │
│  📝 文字规范：自动检查错别字和标点                       │
│  🎭 内容边界：适度情感描写                               │
│  💡 文笔标准：简洁有力，画面感强                         │
└─────────────────────────────────────────────────────────┘

确认以上规范？[Y/n]
```

### 步骤1：项目创建
- 在 `~/novels/[小说名]/` 创建项目
- 生成标准化目录结构

### 步骤2-5：筹备期
- 灵感创意 → 背景设定 → 人物设定 → 故事骨架
- 每步都遵循 novel-standards 规范

### 步骤6：写作控制台
- 调用 novel-writing
- 实时规范检查（错别字、标点、尺度）

### 步骤7：质量验收
- 调用 novel-standards 质量检查清单
- 生成验收报告

### 步骤8：输出成品
- 保存到 `~/novels/[小说名]/成品/`

## 注意事项

1. **规范是基础**：所有创作活动必须遵循 novel-standards 规范
2. **统一存放位置**：作品统一存放在 ~/novels/ 目录
3. **网文风格优先**：写作风格面向网文读者和多媒体改编
4. **实时规范检查**：写作过程中自动检查错别字、标点、尺度
5. **质量验收**：成品输出前必须通过规范检查
