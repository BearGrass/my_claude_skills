# 小说创作工作流引导器

引导用户完成从灵感到成品的完整小说创作流程。

## 工作流程概览

```
┌─────────────────────────────────────────────────────────────────┐
│                     小说创作完整工作流                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  阶段一：筹备期                                                  │
│  ├── 步骤1: 项目创建    →  novel-project                        │
│  ├── 步骤2: 灵感创意    →  novel-ideas                          │
│  ├── 步骤3: 背景设定    →  novel-worldbuilding                  │
│  ├── 步骤4: 人物设定    →  novel-characters                     │
│  └── 步骤5: 故事骨架    →  novel-outline                        │
│                                                                 │
│  阶段二：创作期                                                  │
│  └── 步骤6: 写作控制    →  novel-writing ⭐                     │
│      ├── 初稿写作      →  novel-draft                          │
│      ├── 分层修订      →  novel-revision                       │
│      └── 分层润色      →  novel-polish                         │
│                                                                 │
│  阶段三：成品期                                                  │
│  └── 输出最终稿件                                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 工作模式

### 模式一：完整引导（推荐新手）

适合第一次使用或想要完整指导的用户。

```
阶段一：筹备期
1. 介绍整体流程
2. 询问项目名称，创建项目目录
3. 引导生成创意方案，确认选择
4. 引导构建世界观
5. 引导创建人物设定
6. 引导创作故事大纲

阶段二：创作期（novel-writing 控制）
7. 进入写作控制台
8. 分章节/分层写作初稿
9. 分层修订优化
10. 分层润色提升

阶段三：成品期
11. 输出最终稿件
```

### 模式二：快速启动

适合有经验的作者，快速创建项目结构。

```
1. 获取项目基本信息
2. 创建项目目录
3. 询问要从哪一步开始
4. 直接进入对应阶段
```

### 模式三：继续创作

适合已有项目的用户，继续未完成的工作。

```
1. 检测已有项目目录
2. 分析当前进度
3. 从上次停止的地方继续
```

### 模式四：直接写作

跳过筹备期，直接进入写作阶段。

```
1. 确认已有故事大纲
2. 直接调用 novel-writing
3. 开始写作/修订/润色
```

### 步骤1：项目创建

**目标**：创建标准化的项目目录结构

**执行**：
1. 询问小说项目名称
2. 询问保存位置（默认当前目录）
3. 调用 `novel-project` 技能创建目录
4. 生成项目 README 和各阶段模板

**输出**：完整的项目目录结构

**检查点**：
```

✅ README.md 已生成
✅ 各阶段文件夹已创建

```

### 步骤2：灵感创意

**目标**：生成3-5个小说创意方案供选择

**执行**：
1. 询问是否有初步想法/关键词/类型偏好
2. 调用 `novel-ideas` 技能生成创意
3. 展示所有创意方案
4. 引导用户选择或组合

**输出**：保存到 `01-灵感创意/创意记录.md`

**检查点**：
```
✅ 已生成 X 个创意方案
✅ 用户已选择：[创意名称]

是否继续下一步（背景设定）？[Y/n]
```

### 步骤3：背景设定

**目标**：构建完整的世界观设定

**前置**：读取已选择的创意方案

**执行**：
1. 根据创意类型确定设定框架
2. 通过问答引导完善设定
3. 调用 `novel-worldbuilding` 技能
4. 生成世界观设定文档

**输出**：保存到 `02-背景设定/世界观设定.md`

**检查点**：
```
✅ 时代背景已确定
✅ 世界规则已设定
✅ 核心设定已完善

是否继续下一步（人物设定）？[Y/n]
```

### 步骤4：人物设定

**目标**：创建完整的人物档案

**前置**：读取创意和世界观设定

**执行**：
1. 确定需要塑造的人物列表
2. 优先塑造主角
3. 依次塑造配角和反派
4. 调用 `novel-characters` 技能
5. 建立人物关系网络

**输出**：保存到 `03-人物设定/` 目录

**检查点**：
```
✅ 主角档案已完成
✅ 主要配角档案已完成
✅ 反派档案已完成
✅ 人物关系图已生成

是否继续下一步（故事骨架）？[Y/n]
```

### 步骤5：故事骨架

**目标**：构建完整的故事结构

**前置**：读取创意、世界观、人物设定

**执行**：
1. 确定故事结构模型
2. 规划三幕式框架
3. 细化章节规划
4. 调用 `novel-outline` 技能
5. 确认情节点和节奏

**输出**：保存到 `04-故事骨架/` 目录

**检查点**：
```
✅ 故事大纲已完成
✅ 章节规划已完成（共 X 章）
✅ 关键情节点已确定

筹备期完成！准备进入创作期...
```

## 进度保存与恢复

### 自动保存

每完成一个步骤，自动保存进度到项目目录：

```json
// .novel-progress.json
{
  "project_name": "小说名",
  "current_phase": "创作期",
  "current_step": 6,
  "current_step_name": "写作控制台",
  "completed_steps": [1, 2, 3, 4, 5],
  "last_update": "2026-03-02 12:00:00",
  "writing_progress": {
    "draft_progress": 80,
    "revision_progress": 30,
    "polish_progress": 0
  },
  "stats": {
    "total_chapters": 20,
    "completed_chapters": 15,
    "total_words": 64000
  }
}
```

### 恢复创作

当用户选择"继续创作"模式时：

1. 扫描目录下的项目文件夹
2. 读取 `.novel-progress.json`
3. 显示进度摘要
4. 根据当前阶段跳转到对应位置：
   - 筹备期：跳转到对应步骤
   - 创作期：进入 `novel-writing` 控制台
   - 成品期：显示完成状态

## 交互原则

### 引导风格

1. **渐进式引导**：一次只问一个关键问题
2. **提供选项**：给出建议选项，同时允许自由输入
3. **即时反馈**：每一步都有确认和预览
4. **灵活跳转**：允许跳过或返回前一步

### 确认机制

每个关键步骤完成后：

```
✅ [步骤名称] 已完成

[简要展示成果]

---
• 输入 Y 继续下一步
• 输入 N 暂停，稍后继续
• 输入数字跳转到指定步骤
• 输入 back 返回上一步修改
• 输入 writing 直接进入写作控制台
```

### 异常处理

```
如果用户想跳过某步：
→ 提醒该步骤的重要性
→ 询问是否创建基础版本
→ 允许跳过但标记为"待完善"

如果用户想返回修改：
→ 显示该步骤的当前内容
→ 引导修改
→ 检查对后续步骤的影响

如果用户想暂停：
→ 保存当前进度
→ 告知如何继续
→ 显示恢复指令

如果用户想直接写作：
→ 确认是否已有故事大纲
→ 如有，直接进入 novel-writing
→ 如无，引导先完成筹备期
```

## 注意事项

1. **确保前置技能可用**：此技能需要调用 novel 系列的其他技能
2. **项目目录为工作基础**：所有内容都保存在项目目录中
3. **支持断点续创**：可随时暂停，下次从断点继续
4. **灵活调整**：用户可以不按顺序，跳过或返回任意步骤
5. **写作控制台是核心**：`novel-writing` 整合了初稿、修订、润色，是创作期的主要工具
