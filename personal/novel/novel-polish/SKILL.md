# 小说润色助手

对修订后的小说进行文字层面的精细打磨，输出最终成品。

## 触发前提

此技能应在以下情况使用：
- 用户已完成结构修订（`06-结构修订/`）
- 用户想要进行文字层面的优化
- 用户想要准备最终成品

## 工作流程

### 步骤1：风格定位（使用 AskUserQuestion 工具）

```
问题：请选择作品的风格基调？
选项：
- 网文爽文：快节奏、强冲突、爽感强
- 文艺质感：细腻、有深度、语言精美
- 轻松幽默：诙谐、趣味、轻松愉快
- 沉重深刻：严肃、深刻、有思想深度
- 保持原风：在现有风格基础上优化
```

### 步骤2：选择润色范围（使用 AskUserQuestion 工具）

```
问题：请选择润色范围？
选项：
- 全文润色：对所有章节进行润色
- 重点章节：只润色关键章节
- 指定章节：选择特定章节润色
- 风格统一：只做风格一致性检查
```

### 步骤3：逐章润色

对每章进行精细化打磨

### 步骤4：一致性检查

确保全文风格统一

### 步骤5：最终检验

质量把关

### 步骤6：成品输出

生成最终稿件

## 润色维度

### 一、语言层面

```markdown
## 语言优化检查

### 句子优化
- [ ] 是否有过长的句子需要断句？
- [ ] 是否有句式重复单调？
- [ ] 是否有累赘的修饰语？
- [ ] 句子节奏是否有变化？

### 词语选择
- [ ] 用词是否准确？
- [ ] 是否有重复用词？
- [ ] 是否有更好的表达？
- [ ] 是否符合人物身份？

### 修辞手法
- [ ] 比喻是否恰当新颖？
- [ ] 描写是否生动？
- [ ] 是否有陈词滥调？
- [ ] 是否有过度修辞？

### 常见问题示例

❌ 累赘表达：
他不禁开始感到内心有些许不安的情绪在慢慢蔓延。

✅ 精炼表达：
不安在心底蔓延。

❌ 重复句式：
他看到...他觉得...他决定...他知道...

✅ 变化句式：
看到...的那一刻，他心沉了下去。决定很快做出——...
```

### 二、对话层面

```markdown
## 对话优化检查

### 自然度
- [ ] 对话是否像真人说话？
- [ ] 是否有书面化的口语？
- [ ] 是否有不自然的过渡？

### 人物声音
- [ ] 不同人物是否有不同说话风格？
- [ ] 同一人物对话风格是否一致？
- [ ] 对话是否展现人物性格？

### 功能性
- [ ] 对话是否推动情节？
- [ ] 对话是否展现人物？
- [ ] 对话是否提供信息？

### 对话优化示例

❌ 说明式对话：
"我今年二十八岁，是一名律师，在市中心的一家律师事务所工作。"
"哦，原来如此。你一定是大学毕业了？"
"是的，我五年前从法学院毕业。"

✅ 自然对话：
"在哪高就？"
"混口饭吃，律师。"
"那挺厉害的。"
"还行吧，累。"
```

### 三、描写层面

```markdown
## 描写优化检查

### 感官描写
- [ ] 是否调动多种感官？
- [ ] 描写是否有独特视角？
- [ ] 是否避免陈词滥调？

### 细节选择
- [ ] 细节是否有意义？
- [ ] 是否有过多或过少？
- [ ] 是否有"展示而非告知"？

### 节奏控制
- [ ] 描写是否拖慢节奏？
- [ ] 是否与情节节奏匹配？
- [ ] 是否有"喘息空间"？

### 描写优化示例

❌ 告知式描写：
这是一个阴森的房间，让人感到害怕。

✅ 展示式描写：
墙角的霉斑像一张扭曲的脸。窗户纸破了一半，风灌进来，烛火晃得人心慌。她不敢看床底下——谁知道那里藏着什么。
```

### 四、节奏层面

```markdown
## 节奏优化检查

### 句子节奏
- [ ] 长短句是否有变化？
- [ ] 紧张处是否用短句？
- [ ] 抒情处是否有舒展的长句？

### 段落节奏
- [ ] 段落长短是否有变化？
- [ ] 是否有大段文字需要拆分？
- [ ] 关键时刻是否有独立段落突出？

### 章节节奏
- [ ] 各章篇幅是否均衡？
- [ ] 结尾钩子是否有力？
- [ ] 整体节奏是否有起有伏？

### 节奏优化示例

❌ 平淡节奏：
他走进房间，环顾四周，发现没有人。他松了一口气，坐到沙发上。这时，门突然开了，一个人走了进来。

✅ 变化节奏：
房间里空无一人。
他松了口气，瘫在沙发上。
门——
开了。
```

## 润色流程

### 逐章润色步骤

```
1. 朗读原文
   - 感受节奏和流畅度
   - 标记卡顿或不自然处

2. 句子层面
   - 精简累赘表达
   - 优化句式变化
   - 检查语法错误

3. 段落层面
   - 调整段落结构
   - 优化过渡衔接
   - 检查信息密度

4. 章节层面
   - 检查开篇吸引力
   - 检查结尾钩子
   - 确认整体节奏

5. 风格检查
   - 与前后章风格一致
   - 符合作品整体基调
   - 人物声音一致性
```

## 风格指南

### 不同类型的风格要点

```markdown
## 玄幻/奇幻风格
- 语言可略带古风或诗性
- 修炼/魔法术语统一
- 避免过于现代的网络用语
- 适当使用排比增强气势

## 都市风格
- 语言贴近现实口语
- 可适度使用流行语
- 注重生活细节的真实感
- 对话要符合当代习惯

## 科幻风格
- 科技术语使用准确
- 未来感但不晦涩
- 平衡技术描写与可读性
- 可用简洁有力的短句

## 悬疑风格
- 信息揭示节奏控制
- 氛围营造注重细节
- 句式可简练紧张
- 适当使用省略和留白

## 言情风格
- 情感描写细腻真挚
- 避免过度煽情
- 对话要甜蜜但有个性
- 细节要有心动感
```

## 常见润色技巧

### 精简技巧

```
1. 删除无意义的填充词
   - "就"、"都"、"也"、"其实"、"可以说"...

2. 删除冗余修饰
   - "非常"、"十分"、"极其"等程度副词慎用

3. 删除重复信息
   - 已通过上下文表达的信息不必再说明

4. 删除无效描写
   - 不推动情节、不塑造人物的描写
```

### 增强技巧

```
1. 增加具体细节
   - 抽象→具体：他很紧张→他手心出汗

2. 增加感官描写
   - 视觉以外的感官体验

3. 增加情绪细节
   - 外显行为+内在心理

4. 增加节奏变化
   - 关键时刻用短句强调
```

### 优化技巧

```
1. 句式变化
   - 陈述句+疑问句+感叹句
   - 长句+短句

2. 开头变化
   - 不总是以主语开头
   - 可以时间、地点、动作开头

3. 过渡优化
   - 自然过渡，不用"然后"、"接着"

4. 结尾有力
   - 章节结尾留钩子
   - 段落结尾有力度
```

```markdown
### 基础检验

```

## 输出格式

### ⚠️ 重要原则：优先使用修订稿

```
润色时的稿件来源优先级：
1. 优先：06-结构修订/修订稿/（如果存在修订稿）
2. 其次：05-小说初稿/章节/（如果没有修订稿）

润色后的终稿输出到 07-成品/最终稿/章节/
```

### 目录结构

```
07-成品/
├── 最终稿/
│   ├── 章节/               # ⭐ 分章节版本（主要输出）
│   │   ├── 第1章.md
│   │   ├── 第2章.md
│   │   └── ...
│   └── 全文.md             # 可选：完整合并版（仅当用户需要时生成）
├── 作品信息.md             # ⭐ 必须生成：书名、简介、关键词等
└── 投稿记录.md             # 投稿平台和反馈
```

### ⚠️ 按章节归档的重要性

```
为什么终稿要按章节归档？

1. 避免单文件过长导致的编辑和阅读困难
2. 方便后续修改单个章节
3. 便于投稿时的分段上传
4. 保持与初稿、修订稿的结构一致性

全文.md 是可选的合并版，仅在用户明确需要时生成。
主要输出应该是分章节版本。
```

### 简介写作技巧

```
好的简介应该：
1. 有钩子 - 第一句话就抓住读者
2. 有冲突 - 让读者知道故事的核心矛盾
3. 有悬念 - 让读者想知道结局
4. 有代入感 - 让读者对主角产生兴趣

避免：
- 过多背景设定介绍
- 没有冲突的平淡描述
- 泄露关键剧情转折
```

## 使用示例

**用户输入**：帮我润色小说，准备成品

**处理流程**：
1. 读取修订后的稿件
2. 确认风格定位
3. 逐章进行润色
4. 进行最终检验
5. 输出成品文件

**输出**：完整的成品文件，保存在 `07-成品/` 目录
