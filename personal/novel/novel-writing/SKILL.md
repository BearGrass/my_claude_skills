# 小说写作节奏控制器

整合初稿、修订、润色三个阶段，通过分层处理优化写作节奏，加速创作流程。

## 核心理念

```
传统流程（串行，易阻塞）：
初稿全部 → 修订全部 → 润色全部
❌ 周期长、反馈慢、难以调整

分层流程（并行，快迭代）：
章节 → 单元 → 故事线 → 整体
✅ 快速反馈、灵活调整、持续产出
```

## 分层架构

### 初稿分层

```
┌─────────────────────────────────────────────────────────┐
│                    初稿分层架构                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Level 1: 场景层（最小单元）                             │
│  ├── 单个场景/情节片段                                   │
│  ├── 约 500-2000 字                                     │
│  └── 快速产出，不过度打磨                                │
│                                                         │
│  Level 2: 章节层                                        │
│  ├── 2-5 个场景组成                                     │
│  ├── 约 3000-8000 字                                    │
│  └── 完整的章节叙事                                      │
│                                                         │
│  Level 3: 单元层                                        │
│  ├── 3-10 章组成一个叙事单元                             │
│  ├── 第一幕/第二幕前半/第二幕后半/第三幕                 │
│  └── 具有独立的小高潮和节奏                              │
│                                                         │
│  Level 4: 全文层                                        │
│  └── 整部小说的完整初稿                                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 修订分层

```

│                    修订分层架构                          │
├─────────────────────────────────────────────────────────┤

│  ├── 范围：单章节内                                      │
│  ├── 内容：逻辑、节奏、对话、描写                        │
│  ├── 耗时：约 5-10 分钟/章                               │
│  └── 可并行处理多章                                      │
│                                                         │
│  Level 2: 单元修订（中）                                 │
│  ├── 范围：3-10章的叙事单元                              │

│  └── 确保单元内故事流畅                                  │
│                                                         │
│  Level 3: 故事线修订（慢）                               │
│  ├── 范围：完整故事线（主线/情感线/副线）                │

│  └── 确保故事线完整有力                                  │

│  Level 4: 整体修订（最慢）                               │
│  ├── 范围：全文                                          │
│  ├── 内容：结构平衡、节奏把控、首尾呼应                  │
│  ├── 耗时：约 1-2 小时                                   │
│  └── 最终的全局把控                                      │

└─────────────────────────────────────────────────────────┘
```

## 写作节奏控制

### 节奏模式

#### 模式一：快速迭代（推荐）

适合长篇创作，快速产出，持续优化。

```
循环执行：
┌──────────────────────────────────────┐
│  1. 写一章初稿                        │
│  2. 章节级修订                        │
│  3. 章节级润色                        │
│  4. 标记待优化项                      │
│  5. 继续下一章                        │
└──────────────────────────────────────┘
        ↓ 完成一个单元后
┌──────────────────────────────────────┐
│  6. 单元级修订                        │
│  7. 标记跨章节问题                    │
│  8. 继续写作                          │
└──────────────────────────────────────┘
        ↓ 完成初稿后
┌──────────────────────────────────────┐
│  9. 故事线修订                        │
│  10. 整体修订                         │
│  11. 全文润色                         │
└──────────────────────────────────────┘
```

#### 模式二：阶段推进

适合中短篇，分阶段完成。

```
阶段一：初稿冲刺
├── 快速完成全部初稿
├── 不纠结细节
└── 保持创作热情

阶段二：分层修订
├── 章节修订（快速过一遍）
├── 单元修订（重点打磨）
└── 整体修订（统筹优化）

阶段三：分层润色
├── 问题章节重点润色
└── 全文风格统一
```

#### 模式三：精细打磨

适合短篇精品创作。

```
每章精修：
├── 写完一章
├── 冷却 1 天
├── 修订润色
├── 确认满意
└── 下一章
```

### 继续写作流程

```
1. 检测当前进度
   ├── 读取进度文件
   ├── 确认当前章节和状态
   └── 显示上次完成位置

2. 开始写作
   ├── 回顾章节大纲
   ├── 回顾前后文衔接
   ├── 开始写作/续写
   └── 实时显示字数

3. 完成检查点
   ├── 本节完成？
   ├── 需要快速修订？
   └── 保存并记录进度

4. 选择下一步
   ├── 继续写下一节
   ├── 修订当前内容
   ├── 暂停，稍后继续
   └── 结束本次会话
```

### 章节修订详细流程

```
章节修订（Level 1）：

输入：单个章节文件

检查清单：
├── [ ] 开头是否吸引人？
├── [ ] 结尾是否有钩子？
├── [ ] 对话是否自然？
├── [ ] 描写是否生动？
├── [ ] 节奏是否流畅？
├── [ ] 是否有冗余？
├── [ ] 逻辑是否合理？
└── [ ] 人物是否一致？

输出：
├── 修订后的章节文件
├── 问题清单（需上层修订解决的）
└── 更新进度记录
```

```
单元修订（Level 2）：

输入：3-10章组成的叙事单元

检查清单：
├── [ ] 情节是否连贯？
├── [ ] 人物发展是否合理？
├── [ ] 伏笔是否有呼应？
├── [ ] 节奏是否有起伏？
├── [ ] 是否有遗忘的线索？

└── [ ] 单元高潮是否有力？

输出：
├── 各章节修订建议
├── 跨章节问题清单
├── 需要上层解决的问题
└── 更新进度记录
```

### 故事线修订详细流程

```
故事线修订（Level 3）：

输入：完整的主线/情感线/副线

主线检查：
├── [ ] 核心冲突是否清晰？
├── [ ] 主角目标是否明确？
├── [ ] 障碍是否足够？
├── [ ] 高潮是否有力？
├── [ ] 结局是否满意？
└── [ ] 主题是否深刻？

情感线检查：
├── [ ] 情感发展是否自然？
├── [ ] 是否有化学反应？
├── [ ] 阻碍是否合理？

副线检查：
├── [ ] 是否服务于主线？
├── [ ] 是否有独立价值？
├── [ ] 是否完整闭环？
└── [ ] 比例是否恰当？

输出：
├── 各单元修订建议
├── 故事线问题清单
├── 需要整体层解决的问题
└── 更新进度记录

## 智能建议系统

### 基于进度给出建议

```
当前状态分析：
├── 初稿进度：80%
├── 已完成章节：16/20
├── 写作节奏：正常
└── 预计完成：3天

💡 建议：
1. 今日可完成第17章初稿
2. 建议完成初稿后进行一轮单元修订
3. 第15-16章可以快速过一遍章节修订
```

### 基于问题给出建议

```
问题分析：
├── 第12章：情节跳跃，需要补充过渡
├── 第8章：人物行为突兀，需要铺垫
└── 第5-7章：节奏偏慢，可精简

💡 建议处理顺序：
1. [优先] 第12章补充过渡场景（约30分钟）
2. [优先] 第8章增加心理描写（约20分钟）
3. [次要] 第5-7章精简（可放在整体修订时处理）
```

### 基于时间给出建议

```
当前时间：14:30
预计可用时间：1小时

💡 建议任务：
1. 完成第17章初稿（约40分钟）
2. 快速修订第17章（约15分钟）
3. 更新进度记录（约5分钟）

或者：
1. 单元修订：第15-17章（约45分钟）
2. 更新进度（约5分钟）
```

    └── 调用 → novel-polish（润色技能）
                └── 分层润色支持
```

## 注意事项

1. **分层处理是关键**：避免一次性处理大量内容导致超时或质量下降
2. **保持节奏感**：写-修-润的循环要保持适度节奏
3. **记录问题**：低层修订发现的高层问题要及时记录
4. **灵活调整**：根据实际情况调整分层策略
5. **定期保存**：每个检查点都要保存进度
