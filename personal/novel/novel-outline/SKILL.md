# 小说故事骨架生成器

根据创意、世界观和人物设定，构建完整的故事结构框架。

## 触发前提

此技能应在以下情况使用：
- 用户已有创意灵感（`01-灵感创意/`）
- 用户已有世界观设定（`02-背景设定/`）
- 用户已有人物设定（`03-人物设定/`）
- 用户想要创建故事大纲

## 工作流程

### 步骤1：读取现有素材

读取创意、世界观、人物设定

### 步骤2：选择结构模型（使用 AskUserQuestion 工具）

```
问题：请选择故事结构模型？
选项：
- 经典三幕式：适合大多数故事
- 英雄之旅：适合冒险/成长故事
- 悬疑结构：适合推理/悬疑故事
- 言情结构：适合言情故事
- 自定义结构：根据需要调整
```

### 步骤3：确认篇幅规模（使用 AskUserQuestion 工具）

```
问题：请选择故事篇幅？
选项：
- 短篇（3万字以下）：5-10章
- 中篇（3-15万字）：10-30章
- 长篇（15万字以上）：30章以上
```

### 步骤4：构建整体框架

规划三幕式结构

### 步骤5：细化章节规划

将框架分解为章节

### 步骤6：设计情节点

确定关键转折和高潮

## 故事结构模型

### 经典三幕式结构

```
第一幕：建置（约25%篇幅）
├── 开篇钩子（1-2章）
│   └── 吸引读者，建立兴趣
├── 日常世界（2-3章）
│   └── 展示主角正常生活，暗示不满
├── 激励事件（1章）
│   └── 打破平衡的事件发生
├── 纠葛时刻（1-2章）
│   └── 主角犹豫是否接受冒险
└── 第一幕高潮（1章）
    └── 主角做出决定，跨越门槛

第二幕：对抗（约50%篇幅）
├── 试炼之路（约1/3篇幅）
│   ├── 新世界探索
│   ├── 盟友与敌人登场
│   └── 小胜利与小挫折
├── 中点转折（关键节点）
│   └── 重大事件改变故事方向
├── 危机加深（约1/3篇幅）
│   ├── 赌注提高
│   ├── 内外部压力增大
│   └── 看似绝望的时刻
└── 第二幕高潮
    └── 最大的危机/失去

第三幕：解决（约25%篇幅）
├── 终极对决准备
│   └── 主角重整旗鼓
├── 高潮时刻
│   └── 与反派/困难的最终对决
├── 高潮结局
│   └── 高潮的结果
└── 新平衡
    └── 主角和世界的新的状态
```

### 英雄之旅（适合冒险/成长故事）

```
1. 日常世界
2. 冒险召唤
3. 拒绝召唤
4. 遇见导师
5. 跨越第一道门槛
6. 考验、盟友、敌人
7. 接近最深的洞穴
8. 磨难
9. 奖赏
10. 返回之路
11. 复活
12. 带着灵药归来
```

### 悬疑结构（适合推理/悬疑）

```
第一幕：谜题呈现
├── 案件发生
├── 侦探登场
└── 初步调查

第二幕：调查深入
├── 线索收集
├── 嫌疑人排除
├── 红鲱鱼（误导）
├── 关键发现
└── 真相浮出

第三幕：真相揭示
├── 推理复盘
├── 真相揭晓
└── 动机解释
```

### 言情结构（适合言情）

```
第一幕：相遇
├── 初遇（第一印象）
├── 冲突/吸引
└── 命运绑定

第二幕：发展
├── 接近与退缩
├── 误会与靠近
├── 感情升温
├── 重大冲突/分离
└── 认清心意

第三幕：结局
├── 追寻/挽回
├── 成长与改变
└── 在一起/开放式结局
```

## 情节设计要点

### 关键情节点设计

```markdown
# 情节卡片

## 激励事件
- **是什么**：打破主角日常平衡的事件
- **为什么重要**：开启故事，迫使主角行动
- **设计原则**：必须与主角相关，必须无法忽视

## 第一幕高潮
- **是什么**：主角决定接受冒险的时刻
- **为什么重要**：正式进入故事主体
- **设计原则**：必须是主动选择，不能是被迫

## 中点
- **是什么**：故事方向重大改变的事件
- **为什么重要**：防止故事中段疲软
- **类型**：
  - 正面中点：主角取得重大胜利
  - 负面中点：发现更深的阴谋
  - 身份中点：得知改变认知的真相

## 第二幕高潮
- **是什么**：主角陷入最低谷的时刻
- **为什么重要**：为最终胜利做铺垫
- **设计原则**：必须有希望渺茫的感觉

## 高潮
- **是什么**：与反派的最终对决
- **为什么重要**：故事最高点
- **设计原则**：
  - 必须是主角亲自解决
  - 必须用上成长所得
  - 必须付出代价
```

## 使用示例

**用户输入**：帮我创建故事大纲
